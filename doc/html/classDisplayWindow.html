Detailed Description

User interface class that implements an image processing pipeline.

Constructor & Destructor Documentation

DisplayWindow::DisplayWindow ( String  name)

Simple visualizer constructor.

This constructor produces a window which does no processing. Useful when all that is needed is to display an image stream or to scroll through a folder full of images.

Parameters:
nameName of window to display images in
DisplayWindow::DisplayWindow ( String  name,
std::vector< ProcessingElement * >  prcElm,
std::vector< std::vector< int > >  pipelineVec 
)

Window constructor with processing elements.

This constructor produces a window with a defined set of processing elements arranged into pipelines. The order in which the processing elements are arranged in their own vector doesn't matter - they are applied in the order they are listed in the pipeline vector.

Parameters:
nameName of window to display images in
prcElmVector of processing elements to use.
pipelineVecVector of pipelines

void DisplayWindow::display ( const Mat  image) [virtual]

Displays the passed image until a new image is passed.

This function is fully threadsafe and should be used to push new images as fast as they can be acquired from the image source. The image will be stored, processed and displayed.

Parameters:
imageImage to display and process

void DisplayWindow::setImageFolder ( std::string  dir)

Sets a folder to display images from.

Use this function when the user should be able to advance to the next image manually. The directory is not traversed recursively - only the root directory is searched for images.

Parameters:
dirFull path to directory containing only images or other directories.
